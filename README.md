# Automated Multi-Zone Smart Factory & ASRS Simulation Project
## System Overview

This project simulates a fully automated production line developed using Siemens TIA Portal and Factory I/O.

The system produces a final product by:
- Inspecting incoming raw materials.
- Manufacturing components (lids and bases).
- Buffering production to stabilize flow.
- Assembling components into the final product.

These steps are then followed by palletizing and storing the products.

The architecture is divided into independent automation zones, each responsible for a specific stage of the manufacturing process.
Each zone implements:
- Controlled Start / Stop / Reset behavior
- Emergency handling
- Watchdog timers for fault detection
- Controlled product flow to prevent congestion

HMI device is also configured with industry-standard alarms, users authentication, and user-friendly screens.
Those screens are:
- Overview
- Production
- Alarms
- Maintenance
  
## Zone 1 — Raw Material Feed & Inspection

### Purpose

Zone 1 is responsible for introducing raw materials into the production line and verifying their validity before processing.

Incoming items undergo two inspection steps:

- Weight verification
- Visual verification using a vision sensor
- Items that fail inspection are automatically rejected.

### Process Flow

- A raw item is generated by the feeder.
- The item moves onto the weighing conveyor.
- The conveyor briefly stops when the item reaches the measurement position to obtain a stable weight reading.
- At the same time, the vision sensor verifies the item type.
- The PLC evaluates both conditions.
  
#### Decision Logic

- Weight and vision verification valid	==> Item routed to production
- Weight or vision verification invalid	==> Item routed to discard

Valid items are distributed alternately between the two production lines:
Line A ==> Lid production
Line B ==> Base production
This alternating logic ensures balanced machine utilization.
Rejected items are directed to a discard line.

### Flow Control

To prevent congestion and maintain deterministic behavior:
- Only one item is allowed per production line
- A new raw item is released only when Zone 2 recieves an item.
- The zone includes a reset routine to clear system state after faults

## Zone 2 — Machining

### Purpose

Zone 2 transforms raw material into production components.
Two independent machining lines are implemented:
Line A	==> Lids
Line B	==> Bases

### Process Flow

- Raw material arrives from Zone 1.
- The machine detects the item using an input sensor.
- The machining process is triggered.
- After processing is complete, the component is placed on the output conveyor.
- The finished component moves toward the buffering system in Zone 3.

## Zone 3 — Buffering & Assembly

Zone 3 stabilizes production flow and performs the final product assembly.

This zone consists of two subsystems:
- Component buffering
- Automated assembly

### Buffering System

Each production line contains an identical buffer structure.

#### Buffer Architecture

Each buffer includes:

- 1 gated up elevator
- 3 buffer conveyors
- 3 intermediate gates
- 1 down elevator
Each conveyor section holds 2 components.

Total capacity per buffer:
3 conveyors × 2 items = 6 items

#### Buffer Operation

- Components exit the machining stations.
- Items enter the up elevator, which lifts them to the elevated buffer level.
- The buffer distributes items across the three conveyor sections using gated flow control.
- Each buffer can store up to 6 components.

#### Buffer Synchronization

The two buffers operate in synchronized mode to guarantee correct assembly pairing.

Assembly release conditions:

- At least one lid available
- At least one base available
- Assembler not busy
  
When these conditions are satisfied:

- Both buffers release one item simultaneously.
- Components travel through the down elevators.
- Items arrive at the assembly station.

This synchronization ensures the assembler always receives matched component pairs.

### Assembly Station

The assembly station combines the two components into the final product.

#### Assembly Mechanism

The station includes:

- Two positioners
- One robotic assembler

#### Positioners

Each incoming component is held in place by a positioner to ensure accurate alignment before assembly.

#### Assembly Cycle

- Lid and base arrive at their respective positioners.
- Positioners secure the components.
- The assembler picks the lid component.
- The arm performs two rotation steps to align the part.
- The lid is placed onto the base to create the final product.
- The arm returns to its initial position.
- Positioners lift to release the finished product.
- The product moves toward Zone 4 (Palletizing).

## Zone 4 — Palletizing System

### Purpose

Zone 4 prepares finished products for warehouse storage by grouping them and loading them onto pallets.
The palletizing process ensures that products are organized into transport units before entering the automated storage system.

### Process Flow

- Finished products arrive from the assembly station.
- The palletizer waits until two products are available.
- The products are pushed toward the palletizing drop gate.
- A clamping mechanism holds the products in position.
- The gate opens, allowing the products to fall onto a pallet positioned below.
- This operation only occurs if a pallet is available at the palletizing station.

### Pallet Supply

Pallets are introduced through a lower conveyor system and transported via an elevator.

Pallet preparation sequence:

- A pallet arrives at the palletizer elevator.
- The elevator lifts the pallet to the loading position.
- The system verifies that the pallet is ready to receive products.
  
Only when both conditions are satisfied does pallet loading occur:

- Two finished products available
- Pallet present at loading position

### Pallet Dispatch

After the pallet is loaded with two finished products:

- The elevator lowers the pallet.
- The pallet continues along the conveyor toward Zone 5 – Warehouse Storage.

This creates a consistent palletized transport flow between production and storage.

## Zone 5 — Automated Warehouse & Stacker Crane

### Purpose

Zone 5 implements an automated storage and retrieval system (ASRS) that stores palletized products and retrieves them on demand.
The warehouse is served by a stacker crane capable of placing and retrieving pallets from rack locations.

### Warehouse Capacity

The storage rack contains:

Rack Side	Storage Locations
- Left Rack	54
- Right Rack	54

Total storage capacity:

- 108 pallet locations

### Storage Logic

The warehouse follows a FIFO (First In First Out) strategy.

Storage behavior:

- Incoming pallets are stored in the right rack first.
- If the right rack becomes full, storage continues in the left rack.
  
This storage management is implemented in the PLC using:

- Custom data structures
- Array-based rack representation
- Each storage location is represented as an element within the warehouse memory structure.

### Storage Process

- A pallet arrives from Zone 4.
- The PLC searches for the next available storage location.
- The stacker crane moves to the correct rack position.
- The pallet is placed in the storage location.
- The location is marked as occupied in the warehouse memory structure.

### Retrieval Logic

- The system also supports automated pallet retrieval.
- Retrieval is initiated through the HMI by:
- Specifying the number of pallets required
- Activating the retrieve command

Retrieval process:

- The PLC searches from index 1 for a stored pallet (FIFO).
- The crane moves to the pallet location.
- The pallet is retrieved and delivered to the output conveyor.
- The location is marked as available in the warehouse memory.
- The system continues retrieving pallets until the requested quantity is fulfilled.

## Human Machine Interface (HMI)

The system includes a centralized operator interface developed in Siemens TIA Portal.
The HMI provides monitoring, control, maintenance access, and alarm diagnostics.
User authentication is implemented to restrict access based on user roles.

### HMI Authentication System

The interface supports role-based access levels:

Operator     ==>	Production monitoring and control
Maintenance  ==>	Maintenance and manual control
Admin     	 ==>  Full system access

All screens include:

- Login button
- Logout button
- Current user display

This ensures traceability and controlled system access.

### HMI Screens

### Overview Screen

The Overview screen provides real-time monitoring of the entire factory.

Displayed information includes:

- Zone states and machine status
- Total products currently in production
- Number of items in Buffer A
- Number of items in Buffer B
- Number of occupied warehouse locations
- Graphical bar indicator showing warehouse utilization

### Production Screen

The Production screen allows operators to manage warehouse retrieval orders.

Access Level:

Operator / Maintenance / Admin

Functions:

- Place pallet retrieval orders
- Monitor current order progress
- Cancel active orders
- View statistics

Displayed statistics include:

- Current order quantity
- Remaining pallets to retrieve
- Total orders processed
- Total pallets delivered

### Alarm Screen

The Alarm screen provides system diagnostics.

Features include:

- Active alarms list
- Alarm history log

The alarm system helps identify faults such as:

- Machine timeouts
- Sensor failures
- Process interruptions

This enables faster troubleshooting and system recovery.

### Maintenance Screen

The Maintenance screen provides advanced system control.

Access Level:

Maintenance / Admin only.

Functions include:

- System Reset Tools
- Warehouse memory reset
- Production counter reset
- Manual Equipment Control

Manual control capabilities are provided for:

-Assembly robot
- Stacker crane

Manual operation is only allowed after a safe transition from automatic mode to maintenance mode, ensuring safe machine interaction.

## System Reliability Features

All zones in the system include safety and reliability mechanisms.

### Emergency Handling

Each zone can respond to emergency events by:

- Immediately stopping motion
- Resetting internal machine states
- Safely halting product flow

### Watchdog Monitoring

Each zone contains a watchdog timer that monitors machine state progression.

If a machine remains in the same state longer than expected:

- A fault condition is triggered
- The zone stops operation
- An alarm is generated for the HMI

This prevents situations where machines become stuck due to sensor failure or mechanical blockage.
